How i got the idea to create a static website

This website was created during my semester break. originally I had an unused Raspberry Pi 3 that i wanted to use as a webserver for hosting a website using the Wordpress Content-Management-System.

Long story in brief - I installed LAMP (Linux, Apache, MySQL, PHP) on the pi and found that the performance of the website was not exactly what I had in mind. In retrospect, it’s clear when you consider how many components wordpress consists of.

Looking for an alternative I came across the concept of static website generators. These do not use a CMS which stores the content of the website in a database. Instead, content is pre-generated as HTML code and then loaded onto a server. All files are structured in directories, with a clean separation of content and design. As a result, design changes can easily be made without changing the actual content.

Advantages of static website generators

  1. Speed
    As described above, websites are pre-rendered static HTML. Time-consuming database queries are no longer necessary, so the websites load much faster.

  2. Security
    Static websites use fewer components - where there is no database, no hacking is possible ðŸ˜Š. There is no way to make a malicious request. A user can only request files that contain HTML, CSS, JavaScript or images - the user cannot cause an application to do something malicious or cause a database to reveal privileged information. A CMS with third-party extensions inevitably gives hackers more opportunities for attack.

  3. Version management
    The development process of the website can be easily logged and managed using version controll software such as Git.

  4. Maintenance and handling
    CMS solutions require frequent updates of individual components/ modules. Also, the use of plugins can lead to unpredictable interactions which can be a potential security risk. Static website generators do not require constant updates.

Provider of static website generators

There are a lot of static website generators out there. In the following I would like to introduce two of them in more detail. Both options are easy to use from the command line, have detailed documentation and are easy to use. The choice is purely based on personal preferences.

  1. Jekyll
    Jekyll was founded in 2008 by a co-founder of GitHub and is also used for GitHub Pages. It is one of the most used static website generators. Content is created in Markdown, a language designed for text formatting. Markdown is embedded into an HTML template using the Liquid Template Engine. Jekyll works with libraries like Bootstrap. Existing websites can be migrated to Jekyll using the so-called Importer, for example, you can convert a WordPress page into a Jekyll page.
    To the static website generator Jekyll

  2. Hugo
    Hugo is often compared to Jekyll and is newer. An important feature is its outstanding performance. The content is also written with Markdown and the Template Engine is based on the Go templates. Hugo is written in the Go programming language.
    To the static website generator Hugo

Development process

I would like to keep this section short and refer to the documentation of the providers i have used. The only thing i would like to point out is that i now host this website via GitHub Pages instead of a Raspberry Pi as previously planned. The reasons for this are the combination of version control and hosting provider in one, the ease of use and of course that I still have a Raspberry Pi available ðŸ˜Š.

Summary

In summary, i’m very happy to have found static website generators. I’m also quite happy with my first blog, although it’s already clear to me that i’ll be making some changes to it.
In the future i would like to continue with the design and customazing of hugo themes.
